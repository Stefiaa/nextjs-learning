📁 Next.js File-Based Routing

Learn how Next.js automatically creates routes based on your file structure

How File-Based Routing Works

In Next.js, the file system is used to create routes. Each folder represents a route segment, and each file represents a route. This is called file-based routing.

File Structure Example:

app/
├── page.tsx              → / (home page)
├── routing/
│   ├── page.tsx          → /routing
│   ├── dynamic/
│   │   ├── page.tsx      → /routing/dynamic
│   │   └── [id]/
│   │       └── page.tsx  → /routing/dynamic/[id]
│   └── nested/
│       └── page.tsx      → /routing/nested
└── api/
    └── hello/
        └── route.ts      → /api/hello

Static Routes

Regular folders and files create static routes that are always accessible.

Dynamic Routes

Folders with [brackets] create dynamic routes that can accept parameters.

Route Types Explained

1. Static Routes

Regular folders and files create predictable, always-accessible routes.

/routing/nested

2. Dynamic Routes

Routes with parameters that can change based on user input or data.

/routing/dynamic/[id]

3. Catch-All Routes

Routes that can match multiple segments using [...param] syntax.

/routing/[...slug]

4. Optional Catch-All Routes

Routes that can match multiple segments but are optional using [[...param]].

/routing/[[...slug]]

💡 Key Takeaways

  • File structure = URL structure
  • [brackets] create dynamic routes
  • Use Link component for navigation
  • Routes are automatically created
  • No manual route configuration needed

🧩 React Components in Next.js

Learn how to create and use React components to build your Next.js application

What are React Components?

React components are reusable pieces of UI that can contain their own logic, state, and styling. In Next.js, you use React components to build your pages and create a modular, maintainable codebase.

Component Structure:

function MyComponent() {
  return (
    <div>
      <h1>Hello World</h1>
      <p>This is a React component</p>
    </div>
  )
}

export default MyComponent

Reusable

Create once, use anywhere in your application

Maintainable

Easy to update and debug individual pieces

📡 Data Fetching in Next.js

Learn different ways to fetch and manage data in your Next.js application

Data Fetching Methods

Next.js provides several ways to fetch data depending on when and how you need it.

1. Server Components

Fetch data directly in server components for better performance.

2. API Routes

Create API endpoints in app/api/ directory.

3. Client-side Fetching

Use SWR or React Query for client-side data fetching with caching.

API Route Example

Example API Route:

export default function handler(req, res) {
  res.status(200).json({ 
    message: 'Hello from API!' 
  });
}
